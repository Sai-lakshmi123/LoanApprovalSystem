🛠️ 5 COMPLIANCE & NOTIFICATION TOOLS
════════════════════════════════════════════════════════════════════════════════

Tool 1: record_and_notify ⭐ (Most Useful)
├─ Purpose: Main tool - record action and send all notifications
├─ Inputs: Decision details, applicant info, recipient list
├─ Returns: Case ID, action taken, notifications sent, timestamp, summary
├─ Auto-generates case ID in CASE-YYYYMMDD-XXXXX format
└─ Example: client.record_and_notify("APP001", "John", "john@ex.com", "APPROVE", ...)

Tool 2: get_case_information
├─ Purpose: Retrieve complete case with all notifications
├─ Inputs: Case ID
├─ Returns: Full case record, notification details, audit trail
└─ Example: client.get_case_information("CASE-20260524-00001")

Tool 3: check_notification_status
├─ Purpose: Check delivery status of specific notification
├─ Inputs: Notification ID
├─ Returns: Recipient, status, sent time, delivery confirmation
└─ Example: client.check_notification_status("NOTIF-ABC123...")

Tool 4: get_compliance_report
├─ Purpose: Generate compliance report for all cases
├─ Inputs: Optional start/end dates
├─ Returns: Case counts, action types, notification status summary
└─ Example: client.get_compliance_report()

Tool 5: resend_notification
├─ Purpose: Resend failed notification
├─ Inputs: Case ID, Notification ID
├─ Returns: Resend status and confirmation
└─ Example: client.resend_notification("CASE-20260524-00001", "NOTIF-...")

🔴 CASE ID & NOTIFICATION ID FORMATS
════════════════════════════════════════════════════════════════════════════════

CASE ID:
  Format: CASE-YYYYMMDD-XXXXX
  Example: CASE-20260524-00001

  Components:
    CASE = Fixed prefix
    YYYYMMDD = Date case created (2026-05-24)
    XXXXX = Sequential counter per day (5 digits)

NOTIFICATION ID:
  Format: NOTIF-XXXXXXXXXXXX
  Example: NOTIF-ABC123DEF456

  Components:
    NOTIF = Fixed prefix
    XXXXXXXXXXXX = Unique identifier (hex)

💡 USAGE EXAMPLES
════════════════════════════════════════════════════════════════════════════════

Record Decision and Notify:
  result = client.record_and_notify(
      "APP001", "John Strong", "john@ex.com", "APPROVE",
      1.8, "High", "Balanced", "Strong profile"
  )
  case_id = result['case_id']
  print(f"Notifications sent: {result['notifications_sent']}")

Get Case Details:
  case_info = client.get_case_information("CASE-20260524-00001")
  print(f"Decision: {case_info['case']['decision']}")
  for notif in case_info['notifications']:
      print(f"  {notif['recipient_name']}: {notif['status']}")

Check Notification Status:
  status = client.check_notification_status("NOTIF-ABC123...")
  print(f"Status: {status['status']}")
  print(f"Delivered: {status['delivery_confirmed']}")

Generate Compliance Report:
  report = client.get_compliance_report()
  print(f"Total cases: {report['total_cases']}")
  for action, count in report['cases_by_action'].items():
      print(f"  {action}: {count}")

Resend Notification:
  result = client.resend_notification("CASE-20260524-00001", "NOTIF-...")
  print(f"Resent at: {result['resent_at']}")

Integration Example:

  from mcp.notificationsystem.client import NotificationSystemSyncClient

  decision_db = DecisionSynthesisSyncClient()
  notif_db = NotificationSystemSyncClient()

  # Get decision
  decision = decision_db.synthesize_loan_decision(...)

  # Record and notify
  result = notif_db.record_and_notify(
      applicant_id="APP001",
      applicant_name="John Strong",
      applicant_email="john@example.com",
      decision=decision['decision']['classification'],
      risk_score=decision['decision']['risk_score'],
      confidence=decision['decision']['confidence_level'],
      strategy=decision['strategy_applied'],
      reason=decision['decision']['reasoning']
  )

🆘 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

Connection Refused:
  Make sure server is running: python mcp/notificationsystem/server.py

Case Not Found:
  Check case ID format: CASE-YYYYMMDD-XXXXX (example: CASE-20260524-00001)

Notification Not Found:
  Ensure notification ID exists in system

Timeout:
  Increase client timeout: client.timeout = 60

Port Already in Use:
  Change port in server.py or kill existing process: lsof -i :3003
