🎯 WHAT WAS BUILT
════════════════════════════════════════════════════════════════════════════════

A sophisticated FastMCP server that synthesizes loan decisions from multiple
data sources (Application DB + RiskRulesDB) to produce explainable, compliant
lending decisions with full reasoning and confidence scoring.

Returns:
  ✅ Final Decision (APPROVE/CONDITIONAL/REVIEW/REJECT)
  ✅ Aggregate Risk Score (0-5 scale)
  ✅ Confidence Level (Very High to Very Low)
  ✅ Key Decision Factors (with impact analysis)
  ✅ Detailed Reasoning and Explanation
  ✅ Approval Conditions (for conditional approvals)
  ✅ Scenario Analysis (test different loan amounts)
  ✅ Applicant Comparison (side-by-side analysis)
  ✅ Compliance Audit Trail

🛠️ 4 DECISION SYNTHESIS TOOLS
════════════════════════════════════════════════════════════════════════════════

Tool 1: synthesize_loan_decision ⭐ (Most Useful)
├─ Purpose: Main decision synthesis combining all data sources
├─ Inputs: Applicant data, application analysis, risk assessment, strategy
├─ Returns: Classification, risk score, confidence, factors, reasoning
├─ Strategies: Conservative (strict), Balanced (moderate), Aggressive (relaxed)
└─ Example: client.synthesize_loan_decision("APP001", app_data, app_analysis, risk)

Tool 2: evaluate_multiple_scenarios
├─ Purpose: Compare decisions across different loan amounts
├─ Inputs: Base applicant data, scenario results from RiskRulesDB
├─ Returns: Decision comparison for each scenario
└─ Example: client.evaluate_multiple_scenarios("APP001", ..., scenarios)

Tool 3: explain_decision
├─ Purpose: Get detailed explanation of decision rationale
├─ Inputs: Applicant data, application analysis, risk assessment
├─ Returns: Detailed explanation, key factors, audit trail
└─ Example: client.explain_decision("APP001", app_data, app_analysis, risk)

Tool 4: compare_applicants
├─ Purpose: Compare multiple applicants and their decisions
├─ Inputs: List of applicants with full analyses
├─ Returns: Side-by-side comparison of decisions
└─ Example: client.compare_applicants([app1, app2, app3])

🔴 DECISION STRATEGIES
════════════════════════════════════════════════════════════════════════════════

CONSERVATIVE:
  • Strictest lending criteria
  • DTI < 36%, Credit > 700, Risk < 2.5, No anomalies
  • Lowest approval rate, minimal risk
  • Use Case: Prime lending, risk-averse lenders

BALANCED:
  • Moderate lending criteria
  • DTI < 43%, Credit > 650, Risk < 3.0, Max 1 anomaly
  • Standard approval rate, managed risk
  • Use Case: Conventional loans, standard lending

AGGRESSIVE:
  • Relaxed lending criteria
  • DTI < 50%, Credit > 600, Risk < 3.5, Max 2 anomalies
  • Higher approval rate, managed growth
  • Use Case: Growth-focused, alternative lending


📊 KEY METRICS & SCORING
════════════════════════════════════════════════════════════════════════════════

AGGREGATE RISK SCORE (0-5):
  Component 1: Income Stability (0-5)
  Component 2: Employment Risk (0-5)
  Component 3: Credit Score (0-5)
  Component 4: DTI Ratio (0-5)
  Component 5: Anomalies (0-5)
  Overall = Average of components

CONFIDENCE LEVEL:
  Very High (> 0.85):    Strong data, all metrics agree
  High (0.70-0.85):      Good data quality, minimal concerns
  Moderate (0.55-0.70):  Acceptable data, some concerns
  Low (0.40-0.55):       Limited data or contradictions
  Very Low (< 0.40):     Incomplete data, many concerns

DECISION CLASSIFICATIONS:
  ✅ APPROVE: Issue approval, proceed to closing
  ✅ CONDITIONAL_APPROVE: Approve with specific conditions
  ⚠️ REVIEW: Route to senior underwriter for manual review
  ❌ REJECT: Does not meet lending criteria


🚀 QUICK START
════════════════════════════════════════════════════════════════════════════════

1. Start All Servers (in separate terminals):
   Terminal 1: python mcp/server.py
   Terminal 2: python mcp/riskrulesdb/server.py
   Terminal 3: python mcp/decisionsynthesis/server.py

2. Run Demo:
   python examples/decisionsynthesis_demo.py

3. Use in Code:
   from mcp.decisionsynthesis.client import DecisionSynthesisSyncClient
   from mcp.clients.mcp_client import SyncMCPClient
   from mcp.riskrulesdb.client import RiskRulesDBSyncClient

   app_db = SyncMCPClient()
   risk_db = RiskRulesDBSyncClient()
   decision_db = DecisionSynthesisSyncClient()

   # Get data from all sources
   applicant = app_db.get_applicant_profile("APP001")['data']
   app_analysis = app_db.get_complete_applicant_analysis("APP001")
   risk_assessment = risk_db.generate_risk_report("APP001", applicant, loan)

   # Get decision
   decision = decision_db.synthesize_loan_decision(
       "APP001", applicant, app_analysis['analysis'], risk_assessment
   )

📈 SCORING FORMULAS
════════════════════════════════════════════════════════════════════════════════

Income Stability Component:
  Score = (100 - stability_score) / 100 * 5

Employment Risk Component:
  Score = employment_risk_score / 100 * 5

Credit Score Component:
  750+   → 0.5
  700-749 → 1.5
  650-699 → 2.5
  600-649 → 3.5
  <600    → 4.5

DTI Component:
  <20%   → 0.5
  20-36% → 1.5
  36-43% → 2.5
  43-50% → 3.5
  >50%   → 4.5

Anomaly Component:
  Score = min(anomaly_count * 0.8, 5)

Aggregate Risk Score:
  Final = Average of all 5 components (0-5 scale)

🆘 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

Connection Refused:
  Make sure server is running: python mcp/decisionsynthesis/server.py

Invalid Strategy:
  Use: Conservative, Balanced, or Aggressive

Timeout:
  Increase client timeout: client.timeout = 60

Port Already in Use:
  Change port in server.py or kill existing process: lsof -i :3002

Missing Metrics:
  Ensure all required fields in applicant_data, application_analysis, risk_assessment
