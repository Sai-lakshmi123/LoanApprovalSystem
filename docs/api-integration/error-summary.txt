📊 ERROR CATEGORIZATION STRATEGY
════════════════════════════════════════════════════════════════════════════════

TRANSIENT ERRORS (Retry with backoff):
  • Connection timeout
  • Service unavailable (503)
  • Rate limiting
  • Temporarily unavailable
  • DNS resolution failed
  Result: Automatic retry up to max_retries

PERMANENT ERRORS (Fail immediately, escalate):
  • Invalid API key / Authentication failure
  • Bad request (400)
  • Not found (404)
  • Conflict (409)
  • Invalid data format
  • Business logic violation
  Result: Immediate escalation to manual review

UNKNOWN ERRORS (Retry with caution):
  • Unexpected exceptions
  • Custom error messages
  • Unclassified failures
  Result: Retry with backoff, with built-in limit

✨ FALLBACK MECHANISM
════════════════════════════════════════════════════════════════════════════════

  • Confidence degraded (20-10%)
  • Reasoning: "Decision synthesis failed: [error]"

💡 MONITORING & BEST PRACTICES
════════════════════════════════════════════════════════════════════════════════

Key Metrics to Monitor:
  • Retry rate (should be < 5% of workflows)
  • Fallback decision rate (should be < 1%)
  • Error escalation rate (should be < 2%)
  • Average retries when they occur (should be < 2)

Red Flags:
  ⚠️ Retry rate > 10%
  ⚠️ Fallback rate > 5%
  ⚠️ Persistent errors in same agent
  ⚠️ More permanent than transient errors

Best Practices:
  ✅ Monitor retry metrics daily
  ✅ Increase base_delay for unreliable networks
  ✅ Reduce max_retries for fast-fail scenarios
  ✅ Alert on high escalation rates
  ✅ Review fallback decisions for patterns
  ✅ Test with intentional service failures
