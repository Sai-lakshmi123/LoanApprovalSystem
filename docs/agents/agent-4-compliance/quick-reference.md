## What Agent4 Does

Agent4 records loan decisions, ensures regulatory compliance, and orchestrates notifications to all stakeholders:
- **Decision Recording**: Records in compliance system with case ID
- **Compliance Verification**: Ensures fair lending and regulatory requirements
- **Notification Orchestration**: Sends notifications to appropriate stakeholders
- **Case Management**: Creates case tracking with deadlines and priorities
- **Compliance Documentation**: Generates audit trail and regulatory documentation

**Returns:**
- ✅ Case ID (CASE-YYYYMMDD-XXXXX format)
- ✅ Decision record with compliance verification
- ✅ Notification status and delivery confirmation
- ✅ Compliance certification (fair lending, documentation, audit trail)
- ✅ Case management details (status, next action, assigned to, deadline)
- ✅ Compliance report with key factors and recommendations

---

## Action Classifications

### APPROVE ✅
- **Action**: Record approval, schedule closing
- **Notifications**: Applicant, Loan Officer, Underwriter, Compliance
- **Case Status**: In Progress → Pending Action → Closed
- **Next Steps**: Schedule closing, prepare documents

### CONDITIONAL_APPROVE ✅
- **Action**: Record conditions, request documentation
- **Notifications**: Applicant with conditions, Underwriter, Compliance
- **Case Status**: Open → In Progress → Pending Action
- **Next Steps**: Obtain condition documentation, underwriter approval

### REVIEW ⚠️
- **Action**: Escalate to underwriter, create review task
- **Notifications**: Underwriter, Loan Officer, Compliance
- **Case Status**: Open → In Progress
- **Next Steps**: Manual underwriter review, decision update

### REJECT ❌
- **Action**: Record rejection, prepare appeal information
- **Notifications**: Applicant with appeal process, Loan Officer, Compliance
- **Case Status**: Open → Closed
- **Next Steps**: Appeal window, alternative products, follow-up

## Case ID Format

```
CASE-YYYYMMDD-XXXXX
├─ CASE: Identifier prefix
├─ YYYY: Year (2026)
├─ MM: Month (01-12)
├─ DD: Day (01-31)
└─ XXXXX: Random 5-digit sequence (00000-99999)

Example: CASE-20260525-12345
```

**Characteristics:**
- Immutable once assigned
- Unique within organization
- Includes decision date
- Auto-generated by NotificationSystem

## 5 Available Tools

### Tool 1: record_and_notify
```python
agent.notification_system.record_and_notify(
    applicant_id="APP001",
    applicant_data={...},
    decision_data={...},              # From Agent3
    action_type="decision_notification"
)
```
**Purpose**: Record decision and send initial notifications
**Returns**: Case ID, decision record, notification confirmation

### Tool 2: get_case_information
```python
agent.notification_system.get_case_information(
    case_id="CASE-20260525-12345"
)
```
**Purpose**: Retrieve case details and current status
**Returns**: Complete case information, timeline, actions

### Tool 3: check_notification_status
```python
agent.notification_system.check_notification_status(
    case_id="CASE-20260525-12345"
)
```
**Purpose**: Check notification delivery status
**Returns**: Delivery status per stakeholder, timestamps

### Tool 4: get_compliance_report
```python
agent.notification_system.get_compliance_report(
    case_id="CASE-20260525-12345"
)
```
**Purpose**: Generate compliance documentation
**Returns**: Regulatory report, audit trail, certifications

### Tool 5: resend_notification
```python
agent.notification_system.resend_notification(
    case_id="CASE-20260525-12345",
    stakeholder_type="Applicant"      # Which stakeholder
)
```
**Purpose**: Resend notification to specific stakeholder
**Returns**: Resend confirmation, delivery status

## Configuration

### Change Model
```python
agent = ComplianceActionAgent()
agent.model = "claude-opus-4-7-20250805"     # Most capable
agent.model = "claude-sonnet-4-6-20250514"   # Default
agent.model = "claude-haiku-4-5-20251001"    # Fast
```

### Change NotificationSystem URL
```python
from agents.compliance_action_agent import NotificationSystemClient

agent.notification_system = NotificationSystemClient(
    base_url="http://192.168.1.100:3003"
)
```

### Adjust Timeout
```python
agent.notification_system = NotificationSystemClient(timeout=60)
```

## Common Patterns

### Extract Case ID
```python
result = agent.orchestrate_action(...)
case_id = result["result"]["case_id"]
print(f"Case: {case_id}")
```

### Check Notification Status
```python
result = agent.orchestrate_action(...)
case_id = result["result"]["case_id"]

status = agent.notification_system.check_notification_status(case_id)
for delivery in status["delivery_status"]:
    print(f"{delivery['stakeholder']}: {delivery['status']}")
```

### Verify Compliance
```python
compliance = result["result"]["compliance_certification"]
if compliance["fair_lending_compliant"]:
    print("✅ Fair lending compliant")
else:
    print("❌ Compliance issue detected")
```

### Get Next Steps
```python
case_mgmt = result["result"]["case_management"]
print(f"Status: {case_mgmt['status']}")
print(f"Next Action: {case_mgmt['next_action']}")
print(f"Deadline: {case_mgmt['deadline']}")
print(f"Priority: {case_mgmt['priority']}")
```

### Resend Notification
```python
result = agent.orchestrate_action(...)
case_id = result["result"]["case_id"]

# Resend to applicant if notification failed
resend = agent.notification_system.resend_notification(
    case_id,
    "Applicant"
)
```

## Integration with Full Pipeline

```python
from agents.application_profile_agent import ApplicationProfileAgent
from agents.financial_risk_analysis_agent import FinancialRiskAnalysisAgent
from agents.loan_decision_agent import LoanDecisionAgent
from agents.compliance_action_agent import ComplianceActionAgent

# Step 1: Profile analysis
profile_agent = ApplicationProfileAgent()
profile = profile_agent.analyze_applicant("APP001")

# Step 2: Risk analysis
risk_agent = FinancialRiskAnalysisAgent()
risk = risk_agent.analyze_financial_risk("APP001", applicant_data, loan_request)

# Step 3: Decision synthesis
decision_agent = LoanDecisionAgent()
decision = decision_agent.make_decision(
    "APP001",
    applicant_data,
    profile["analysis"],
    risk["analysis"]
)

# Step 4: Compliance & action orchestration
action_agent = ComplianceActionAgent()
action = action_agent.orchestrate_action(
    "APP001",
    applicant_data,
    decision["decision"]
)

print(f"Case ID: {action['result']['case_id']}")
print(f"Status: {action['result']['case_management']['status']}")
print(f"Next Steps: {action['result']['next_steps']}")
```

## Error Handling

### Connection Error
```
Error: Connection refused
→ Start NotificationSystem server
  python mcp/notificationsystem/server.py
```

### Timeout Error
```
Error: Request timeout
→ Increase timeout in NotificationSystemClient
  client = NotificationSystemClient(timeout=60)
```

### Invalid Case ID
```
Error: Case ID not found
→ Verify case was successfully recorded
→ Check get_case_information returned valid ID
```

### Notification Failed
```
Error: Notification delivery failed
→ Check stakeholder contact information
→ Use resend_notification tool
→ Escalate to compliance officer
```
