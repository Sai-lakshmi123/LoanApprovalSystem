🎯 WHAT WAS BUILT
════════════════════════════════════════════════════════════════════════════════

An intelligent loan application analyzer powered by Claude Sonnet 4.6 and built
with the Anthropic Agent SDK. Agent1 analyzes loan applicants comprehensively
and returns structured analysis with four key metrics.

Returns:
  ✅ Income Stability Score (0-100 with category)
  ✅ Employment Risk Assessment (0-100 with level)
  ✅ Credit History Summary (with score category)
  ✅ Application Completeness Flags (with percentage)
  ✅ Overall Assessment and Recommendations
  ✅ Key Strengths and Concerns
  ✅ Structured JSON Output

🔧 6 TOOLS AVAILABLE
════════════════════════════════════════════════════════════════════════════════

Tool 1: get_applicant_profile
  Purpose: Fetch complete applicant data
  Returns: Name, age, income, employment, credit info

Tool 2: get_income_stability_score
  Purpose: Analyze income stability with score (0-100)
  Returns: Score, category, detailed analysis

Tool 3: get_employment_risk
  Purpose: Assess employment risk level
  Returns: Risk score, level, tenure, type

Tool 4: get_credit_history_summary
  Purpose: Review credit profile and history
  Returns: Score, category, delinquencies, utilization

Tool 5: check_application_completeness
  Purpose: Verify all required fields present
  Returns: Completeness %, missing items, flags

Tool 6: get_complete_applicant_analysis
  Purpose: Get all metrics in single call
  Returns: Complete analysis with all metrics


📊 SYSTEM PROMPT
════════════════════════════════════════════════════════════════════════════════

The agent operates with a detailed system prompt that instructs Claude to:

ROLE:
  Expert loan application profile analyst

ANALYSIS FRAMEWORK:
  1. Income Stability (0-100)
     - Categories: Very Unstable (0-30), Unstable (31-50),
                   Stable (51-70), Very Stable (71-100)
     - Evaluates: Sources, consistency, growth, variability

  2. Employment Risk (0-100)
     - Levels: Very Low (0-20), Low (21-40), Medium (41-60),
              High (61-80), Very High (81-100)
     - Evaluates: Type, tenure, industry, stability

  3. Credit History Summary
     - Score categories: Excellent (750+), Good (700-749),
                        Fair (650-699), Poor (600-649), Very Poor (<600)
     - Includes: Delinquencies, defaults, utilization, age, mix

  4. Application Completeness
     - Percentage: 0-100%
     - Identifies: Missing items, incomplete sections
     - Flags: Items requiring follow-up

GUIDELINES:
  - Gather data systematically
  - Analyze thoroughly and objectively
  - Provide clear reasoning for all scores
  - Flag concerns and discrepancies
  - Highlight strengths and mitigating factors
  - Ensure fair lending compliance
  - Use actual data, not assumptions

OUTPUT FORMAT:
  Always return valid JSON with exact fields:
  - applicant_id
  - applicant_name
  - analysis_timestamp
  - income_stability {...}
  - employment_risk {...}
  - credit_history {...}
  - application_completeness {...}
  - overall_assessment
  - key_strengths []
  - key_concerns []
  - recommended_next_steps []

💡 USAGE EXAMPLE
════════════════════════════════════════════════════════════════════════════════

from agents.application_profile_agent import ApplicationProfileAgent

# Initialize agent
agent = ApplicationProfileAgent()

# Analyze applicant
result = agent.analyze_applicant("APP001")

if result["status"] == "success":
    analysis = result["analysis"]

    # Access structured output
    print(f"Applicant: {analysis['applicant_name']}")
    print(f"Income Stability: {analysis['income_stability']['score']} ({analysis['income_stability']['stability_category']})")
    print(f"Employment Risk: {analysis['employment_risk']['risk_level']} ({analysis['employment_risk']['risk_score']})")
    print(f"Credit Score: {analysis['credit_history']['credit_score']} ({analysis['credit_history']['score_category']})")
    print(f"Completeness: {analysis['application_completeness']['completeness_percentage']}%")
    print(f"\nAssessment: {analysis['overall_assessment']}")

🏗️ ARCHITECTURE
════════════════════════════════════════════════════════════════════════════════

Agent Execution Loop:

1. INITIALIZE
   - Load system prompt
   - Define 6 tools
   - Connect to Application DB

2. SEND REQUEST
   - User input: "Analyze APP001"
   - Include system prompt
   - Include tool definitions

3. CLAUDE PROCESSES
   - Understands task
   - Identifies needed data
   - Decides which tools to call

4. TOOL USE ITERATION
   - Tool calls made
   - Results returned to Claude
   - Claude analyzes results
   - May call more tools

5. SYNTHESIZE
   - Claude analyzes all data
   - Creates structured JSON
   - Includes assessments and recommendations

6. RETURN RESULT
   - Parsed JSON returned
   - Status: "success"
   - All required fields populated

🚨 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

Issue: Connection Refused
  Cause: Application DB server not running
  Solution: python mcp/server.py

Issue: Invalid API Key
  Cause: ANTHROPIC_API_KEY not set
  Solution: export ANTHROPIC_API_KEY=sk-...

Issue: JSON Parse Error
  Cause: Claude output format invalid
  Solution: Check response, may need prompt adjustment

Issue: Tool Not Found
  Cause: Application DB missing tool
  Solution: Verify all 6 tools available in Application DB

Issue: Timeout
  Cause: Slow database or network
  Solution: Increase timeout: ApplicationDBClient(timeout=60)
