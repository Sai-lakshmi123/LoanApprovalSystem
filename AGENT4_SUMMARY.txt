🎯 WHAT WAS BUILT
════════════════════════════════════════════════════════════════════════════════

An intelligent compliance and action orchestrator powered by Claude Sonnet 4.6
and built with the Anthropic Agent SDK. Agent4 records final loan decisions,
ensures regulatory compliance, and orchestrates notifications to all stakeholders.

Returns:
  ✅ Case ID (CASE-YYYYMMDD-XXXXX format, immutable)
  ✅ Decision Record (Classification, Risk Score, Compliance Verified)
  ✅ Notifications (Sent to appropriate stakeholders, delivery tracked)
  ✅ Compliance Certification (Fair lending, documentation, regulatory, audit)
  ✅ Case Management (Status, next action, assigned to, deadline, priority)
  ✅ Compliance Report (Decision summary, key factors, regulatory notes)
  ✅ Action Summary (What was accomplished, escalations if any)
  ✅ Structured JSON Output

🔧 5 TOOLS AVAILABLE
════════════════════════════════════════════════════════════════════════════════

Tool 1: record_and_notify
  Purpose: Record decision and send initial notifications
  Input: applicant_id, applicant_data, decision_data, action_type
  Returns: Case ID, decision record, notification status

Tool 2: get_case_information
  Purpose: Retrieve case details and current status
  Input: case_id (CASE-YYYYMMDD-XXXXX format)
  Returns: Complete case information, timeline, actions

Tool 3: check_notification_status
  Purpose: Check notification delivery status
  Input: case_id
  Returns: Delivery status per stakeholder, timestamps

Tool 4: get_compliance_report
  Purpose: Generate compliance documentation
  Input: case_id
  Returns: Regulatory report, audit trail, certifications

Tool 5: resend_notification
  Purpose: Resend notification to specific stakeholder
  Input: case_id, stakeholder_type
  Returns: Resend confirmation, delivery status


📊 SYSTEM PROMPT
════════════════════════════════════════════════════════════════════════════════

The agent operates with a 140+ line system prompt that instructs Claude to:

ROLE:
  Compliance and action orchestrator recording decisions and notifications

KEY RESPONSIBILITIES:
  1. Decision Recording (Immediate, with case ID generation)
  2. Compliance Verification (Fair lending, documentation, regulatory)
  3. Notification Orchestration (Stakeholder selection per decision)
  4. Case Management (Status tracking, deadlines, priorities)
  5. Compliance Documentation (Audit trail, regulatory docs)
  6. Stakeholder Communication (Appropriate notifications)
  7. Status Tracking (Delivery status, escalations)

STAKEHOLDER TYPES (6):
  1. Applicant: Decision + next steps
  2. Loan Officer: Summary + action items
  3. Underwriter: Details + conditions
  4. Compliance Officer: Certification + documentation
  5. Management: Portfolio metrics + summary
  6. Legal: Regulatory docs + appeal process

ACTION CLASSIFICATIONS (4):
  APPROVE: Full approval → Schedule closing
  CONDITIONAL_APPROVE: Conditions required → Request documentation
  REVIEW: Manual review needed → Escalate to underwriter
  REJECT: Denial → Provide appeal process

CASE ID FORMAT:
  CASE-YYYYMMDD-XXXXX
  - Immutable once assigned
  - Unique within organization
  - Includes decision date
  - Random 5-digit sequence

NOTIFICATION STRATEGY:
  Decision-based stakeholder selection
  - APPROVE: Applicant, Loan Officer, Underwriter, Compliance, Management
  - CONDITIONAL: Applicant, Underwriter, Compliance, Management
  - REVIEW: Underwriter, Loan Officer, Compliance, Management
  - REJECT: Applicant, Loan Officer, Compliance, Legal, Management

OUTPUT FORMAT:
  Valid JSON with exact required fields
  - applicant_id, case_id, action_timestamp
  - decision_record (classification, risk_score, compliance_verified)
  - notifications (sent_to, total_notifications, delivery_status)
  - compliance_certification (fair_lending, documentation, regulatory, audit)
  - case_management (status, next_action, assigned_to, deadline, priority)
  - compliance_report (decision_summary, key_factors, regulatory_notes)
  - action_summary, next_steps, escalations

💡 USAGE EXAMPLE
════════════════════════════════════════════════════════════════════════════════

from agents.compliance_action_agent import ComplianceActionAgent

agent = ComplianceActionAgent()

result = agent.orchestrate_action(
    applicant_id="APP001",
    applicant_data={
        "name": "John Strong",
        "email": "john.strong@example.com",
        "phone": "+1-555-0100",
        "annual_income": 150000,
        "credit_score": 760
    },
    decision_data={
        "decision": {
            "classification": "APPROVE",
            "risk_score": 1.8,
            "confidence_percentage": 92,
            "reasoning": "Strong profile"
        },
        "compliance_notes": {
            "fair_lending_compliant": True,
            "documentation_complete": True
        }
    }
)

if result["status"] == "success":
    action = result["result"]
    print(f"Case ID: {action['case_id']}")
    print(f"Status: {action['case_management']['status']}")
    print(f"Notifications: {action['notifications']['total_notifications']}")
    print(f"Compliance: {action['compliance_certification']['fair_lending_compliant']}")

🚨 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

Issue: Connection Refused
  Cause: NotificationSystem server not running
  Solution: python mcp/notificationsystem/server.py

Issue: Invalid API Key
  Cause: ANTHROPIC_API_KEY not set
  Solution: export ANTHROPIC_API_KEY=sk-...

Issue: Case ID not found
  Cause: Case not successfully recorded
  Solution: Check record_and_notify result, verify case_id

Issue: Notification delivery failed
  Cause: Stakeholder contact information invalid
  Solution: Use resend_notification tool, check contact info

Issue: Timeout
  Cause: Slow server or network
  Solution: Increase timeout: NotificationSystemClient(timeout=60)
